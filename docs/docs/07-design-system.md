# 07 · Design System

## Name
Waykeeper

## Design thesis
Waykeeper is a calm, premium daily planner that feels like a modern editorial folio held inside a softly atmospheric world. It combines:
- soft editorial minimalism
- wellness-tech clarity
- Scandinavian restraint
- subtle sword-and-sorcery codex / wayfinding motifs

## Design goals
- Calm, not bland
- Elegant, not precious
- Distinctive, not theatrical
- Legible, not dashboard-heavy

## Material system
### 1. Paper surfaces
For main planning surfaces. Matte, pale, editorial.

### 2. Frosted air / veil surfaces
For atmosphere and supporting panels. Airy, luminous, cooler.

### 3. Dark shell surfaces
Used sparingly for structure and emphasis. Think obsidian / inkstone / lacquer.

## Color roles
- Atmospheric background tones
- Paper tones
- Accent tones
- One dark shell tone

Avoid candy pastel palettes or corporate SaaS blues.

## Background art direction
Background treatments should stay style-first rather than illustrative or scene-dependent.

Use:
- saturated, painterly, soft-focus impressionist texture
- luminous color blending
- blurred edges
- layered pigment feel
- atmospheric, diffused color fields

Avoid:
- literal landscape illustration
- crisp photoreal detail
- sharp fantasy scene painting
- busy narrative imagery behind the planner

## Typography
- Main UI: clean sans
- Select headings / ceremonial moments: restrained serif

Do not use faux fantasy fonts.

## Iconography
Allowed motifs:
- waypoints
- compass arcs
- lanterns
- stars
- dragon/serpent glyphs
- tiny chest/relic markers
- vines / seasonal marginalia

All iconography should stay SVG, simplified, and modern.

## Ornament rules
Use fantasy influence as residue, not costume.

Good:
- subtle banner corner motifs
- route-line geometry
- small glyphs
- seasonal marginalia

Bad:
- quest log framing
- parchment spam
- dragons everywhere
- overdone medieval styling

## Motion rules
- Slow
- Soft
- Deliberate
- Spatially legible

Avoid bouncy gamified motion.

## Oracle surface behavior
Oracle should feel alive through contextual statefulness, not through animation spam or chatty copy.

Use:
- concise state shifts
- an atmospheric backdrop layer with very light day-part tinting
- restrained emphasis when the current block changes
- brief foregrounding of change summaries after meaningful planner events
- smooth deck transitions between Oracle states
- premium, quiet panel behavior that supports the route instead of competing with it

Avoid:
- assistant bubbles
- typing theatrics
- ornamental motion that makes the side surface feel like a chatbot
- permanent dashboard clutter
- literal window scenes or decorative Oracle props before the interaction model is fully settled

## Copy tone rules
The product voice should be:
- calm
- exact
- slightly elevated
- non-chatty

Prefer:
- arrange
- revise
- place
- move
- defer
- allocate
- remaining
- current block
- next block

Avoid accidental innuendo or therapy-speak.

## Product feel
Waykeeper should feel like a tool for reading and revising the day’s path through chaos.
The route stays central; Oracle should feel like the calm side surface that helps the user read what matters without turning the app into a character experience.
